## 4. Deployment Checklist

### Prerequisites ✅
- [x] PowerShell 5.1+ installed
- [x] Execution policy set to `Bypass` or `RemoteSigned`
- [x] All modules paths configured correctly
- [x] Network connectivity for CI/CD and notifications

### Pre-Deployment Steps
```powershell
# 1. Validate module paths
$paths = @(
    'D:\lazy init ide\RawrXD.Config.psm1',
    'D:\lazy init ide\RawrXD.Logging.psm1',
    'D:\lazy init ide\RawrXD.Core.psm1'
)
$paths | ForEach-Object { Write-Host "$(Test-Path $_): $_" }

# 2. Run analyzer to verify all methods
& 'D:/lazy init ide/auto_generated_methods/Analyze_AutoGeneratedMethods.ps1'

# 3. Run universal integrator
Import-Module 'D:/lazy init ide/auto_generated_methods/RawrXD_UniversalIntegrator.psm1' -Force
Invoke-RawrXD_UniversalIntegration
```

### Deployment Steps

**Option 1: Manual Deployment**
```powershell
# Import all modules
Import-Module 'D:\lazy init ide\RawrXD.Config.psm1' -Force
Import-Module 'D:\lazy init ide\RawrXD.Logging.psm1' -Force
Import-Module 'D:\lazy init ide\RawrXD.Core.psm1' -Force
Import-Module 'D:\lazy init ide\auto_generated_methods\RawrXD_UniversalIntegrator.psm1' -Force

# Execute integration
Invoke-RawrXD_UniversalIntegration -AutoMethodsDir 'D:/lazy init ide/auto_generated_methods'

# Monitor logs
Get-Content (Get-RawrXDLogPath) -Tail 50 -Wait
```

**Option 2: CI/CD Deployment**

## 7. Rollback Plan

If production issues occur:

```powershell
# 1. Stop all background jobs
Get-Job | Stop-Job

# 2. Clear event subscriptions
Get-EventSubscriber | Unregister-Event

# 3. Remove modules from session
Get-Module RawrXD* | Remove-Module -Force

# 4. Restore previous configuration (if backed up)
Copy-Item 'D:/lazy init ide/config/RawrXD.config.json.backup' -Destination 'D:/lazy init ide/config/RawrXD.config.json'

# 5. Restart PowerShell session
$host.SetShouldExit(0)
```

## 8. Support & Troubleshooting

### Common Issues

**Issue**: Module import fails with circular dependency  
**Resolution**: Ensure modules are imported in order: Config → Logging → Core

**Issue**: Manifest validation fails  
**Resolution**: Verify JSON syntax in manifest files using `ConvertFrom-Json`

**Issue**: Notifications not being sent  
**Resolution**: Check network connectivity and verify endpoints in config

**Issue**: Tests failing  
**Resolution**: Review test_report.json for specific test failures and error messages
