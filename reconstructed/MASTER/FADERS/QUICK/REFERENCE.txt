## State Variables

```
g_bFaderDragging     = 0 (idle), 1 (volume drag), 2 (pan drag)
g_nFaderStartX       = Mouse X position when drag started
g_fFaderStartValue   = Fader value when drag started
g_nFaderHover        = -1 (none), 0 (volume), 1 (pan)
```

---

## Message Flow Summary

```
Mouse Down → Detect click on fader area
   ↓
Store drag state (start X, start value)
   ↓
Mouse Move → Calculate delta from start X
   ↓
Update fader value real-time
   ↓
Apply to audio (volume multiplier or pan gain)
   ↓
Mouse Up → Clear drag state
```

---

## Audio Impact

### Volume
```
Original audio × g_fMasterVolume
Range: 0.0 (silent) to 1.0 (unchanged)
```

### Pan
```
Left = original_left × (1 - pan) × 0.5
Right = original_right × (1 + pan) × 0.5

Where pan ranges from -1.0 (full left) to +1.0 (full right)
Center (0.0) = equal gain both channels
```

## Testing Quick Checks

1. **Volume Drag**: Click fader, drag left → volume drops, right → increases
2. **Pan Drag**: Click fader, drag left → "LEFT", center → "C", right → "RIGHT"
3. **Hover**: Move mouse over fader → knob turns orange
4. **Drag**: Hold mouse, drag → knob turns yellow
5. **Release**: Release mouse → knob returns to gray
6. **Audio**: Volume 0% → silent, 100% → full volume
7. **Stereo**: Pan left → sound left channel, right → right channel

## Common Issues & Fixes

| Issue | Cause | Fix |
|-------|-------|-----|
| Fader won't drag | Click outside fader area | Click directly on gray knob |
| Value doesn't update | Drag too slow | Use longer drag motion |
| Audio still loud | Check volume not at max | Verify volume display < 100% |
| Pan not balanced | Pan not exactly centered | Move to "C" display |
| Color doesn't change | Screen not refreshing | Move mouse to trigger redraw |
