REG ADD "HKCU\Software\Google\Chrome\NativeMessagingHosts\com.my_company.my_application" /ve /t REG_SZ /d "<path-to-manifest.json>" /f
native-host-manifest.json
To properly run that command, you need to replace <path-to-manifest.json> with the actual full path to your native-host-manifest.json file. The command is not meant to be run literally with the placeholder name.
Step-by-step instructions
Find the path to your manifest file.
Find the native-host-manifest.json file on your computer. It should be located with your compiled native host executable.
Right-click the file and select Properties.
In the Properties window, copy the entire path from the Location field.
Append \native-host-manifest.json to the path you just copied.
Example: C:\Users\YourUser\source\repos\AIDesktopHelper.Host\bin\Debug\net8.0\native-host-manifest.json
Open Command Prompt as an administrator.
Click the Start button.
Type "Command Prompt" or "cmd."
Right-click the Command Prompt app and select Run as administrator.
Run the command with the correct path.
In the Command Prompt window, type or paste the REG ADD command.
Replace <path-to-manifest.json> with the full path you found in step 1.
The command should look similar to this:
batch
REG ADD "HKCU\Software\Google\Chrome\NativeMessagingHosts\com.my_company.my_application" /ve /t REG_SZ /d "C:\Users\YourUser\source\repos\AIDesktopHelper.Host\bin\Debug\net8.0\native-host-manifest.json" /f
Use code with caution.

Press Enter to execute the command. You should see a message indicating that the operation was completed successfully. 
This process registers your native messaging host with the Chrome browser so that your extension can communicate with it.
