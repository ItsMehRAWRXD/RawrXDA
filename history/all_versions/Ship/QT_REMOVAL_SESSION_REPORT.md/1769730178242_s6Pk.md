## 💡 Lessons Learned

### What Worked Well
- **Consistent Build System**: Using identical compiler flags across all components
- **Onecore Libraries**: Provides clean separation from legacy Windows APIs
- **STL Over Qt Containers**: std::vector, std::map, std::string work perfectly
- **Custom Event System**: More flexible than Qt signals/slots
- **Incremental Approach**: Building one component at a time ensures quality

### Challenges Overcome
- **CRT Linking**: Resolved by using msvcrt.lib with onecore approach
- **std::min Conflicts**: Replaced with manual comparison to avoid macro issues
- **Win32 API Naming**: Avoided conflicts by renaming methods (GetCurrentDir vs GetCurrentDirectory)
- **JSON Parsing**: Built minimal parser instead of heavy external library

### Best Practices Established
- Always use UNICODE/UTF-16 internally
- Wrap Win32 APIs in RAII classes where possible
- Use std::wstring for all text to match Win32 APIs
- Log extensively with OutputDebugStringW
- Export C interface for DLL functions
