## Known Limitations & Trade-offs

1. **Signal/Slot Removal**: Qt's elegant signal/slot system is replaced with direct function calls. More verbose but still clean.

2. **Threading Model Change**: QThread → std::thread. Simpler but requires manual sync management (std::mutex, std::condition_variable).

3. **Removed QSettings**: Need to implement custom settings management via Win32 registry or INI files.

4. **Removed Qt Container Classes**: QString, QList, QMap replaced with std::string, std::vector, std::unordered_map.

5. **UI Rendering**: If UI was entirely Qt-based, will need fallback to GDI+ or DirectWrite for text rendering.
