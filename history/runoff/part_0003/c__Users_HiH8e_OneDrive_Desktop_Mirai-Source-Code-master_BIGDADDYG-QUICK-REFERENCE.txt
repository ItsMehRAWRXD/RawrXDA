# 🚀 BigDaddyG Launcher - Quick Reference Card

## ONE-COMMAND START

```batch
START-BIGDADDYG.bat
```

Or from PowerShell:
```powershell
.\bigdaddyg-launcher-interactive.ps1
```

---

## INTERACTIVE MENU (Every Time)

### Step 1: Choose Model
```
[1] 36 GB model    ← Full power, best accuracy
[2] 36 GB backup
[3] 0.5 GB code    ← Fast, specialized
[0] Exit
```

### Step 2: Configure Settings
```
Inference Mode:    CPU / GPU CUDA / GPU ROCm
Service Port:      Default (8765) / Custom
Memory Limit:      Auto / 4GB / 8GB / 16GB / 32GB
CPU Threads:       Auto / Half / Custom
Log Level:         Info / Debug / Trace
API Auth:          None / Token-based
```

### Step 3: Review & Confirm
```
Shows all settings ↓
Proceed? (yes/no)
```

### Step 4: Service Launches
```
✅ Ready at http://localhost:8765
```


```powershell

```


## INTEGRATE WITH CYBERFORGE

After launcher shows integration prompt, add to `cyberforge-av-engine.js`:

```javascript
const response = await fetch("http://localhost:8765/api/classify", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ file_path: filePath })
});

const result = await response.json();
console.log(`Classification: ${result.classification}`);
```

## KEY POINTS

✅ **Different settings every run** - Menu appears each time  
✅ **Flexible configuration** - Choose CPU/GPU, memory, port, etc.  
✅ **Auto-generated scripts** - Launcher creates startup script on-the-fly  
✅ **Logged activity** - All runs tracked in `logs/` folder  
✅ **Easy to test** - Quick to switch models and settings

## TROUBLESHOOTING

| Issue | Fix |
|-------|-----|
| Can't find models | Check `D:\BigDaddyG-Standalone-40GB\` exists |
| Python not found | Install Python 3.8+, add to PATH |
| Port in use | Select different port in menu [Step 2b] |
| Out of memory | Select smaller model or lower memory limit |
| Slow inference | Switch from CPU to GPU [Step 2a] |
