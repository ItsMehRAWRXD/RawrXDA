## 🔐 Security Model

### Encryption
- **Master Seed**: Generated once, stored in Windows Credential Store
- **Per-User Keys**: HKDF(userId, master_seed) → 256-bit AES key
- **Cipher**: AES-256-GCM (NIST approved)
- **Auth Tag**: 128-bit (detects tampering)
- **IV**: 96-bit random per message

### Access Control
| Role | Permissions |
|------|-------------|
| OWNER | read, write, agentic, index, delete |
| CONTRIBUTOR | read, write, agentic |
| READER | read only |

### Audit Trail
Every mutation (store, update, delete, grant) creates immutable entry:
```
timestamp, user_id, action, detail, hash(entry + previous_hash)
```
SHA-256 chain prevents tampering.
