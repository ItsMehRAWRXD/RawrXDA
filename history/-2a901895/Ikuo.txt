Phase-4 Ultra Kit\n==================\n\nThis folder contains a self-contained browser-only IDE + harness wired to your 4.7 GB GGUF model.\n\nFiles:\n  - Phase4-Ultra.html      : Minimal PR8 mmap/WebGPU harness with metrics + D: drive context search.\n  - HyperIDE-WASM.html     : Full Monaco-based Hyper-IDE wired to the same llama worker.\n  - hyper-ide-main.js      : Boots Monaco, Rawr overlay, and the llama worker for Hyper-IDE.\n  - llama-worker.js        : PR8-backed llama.cpp worker used by Hyper-IDE.\n  - llama.runtime.js       : PR8-style runtime loader (used by alternative builds).\n  - syntax-checker.js      : Local Node script to sanity-check embedded <script> blocks.\n\nYou must place your compiled llama.wasm next to these files. The worker expects exports like:\n  _malloc, _free, _write_file, _llama_init_from_file,\n  _llama_generate, _llama_next_token, _llama_token_to_piece, _llama_free_iterator, etc.\n\nQuick start:\n  1. Copy your 4.7 GB GGUF model somewhere accessible.\n  2. Place llama.wasm in this folder.\n  3. Open Phase4-Ultra.html in a modern browser (Chrome/Edge).\n  4. Click "Mount 4 GB GGUF" and choose your GGUF file.\n  5. Click "Index D: drive" and choose the root folder on D: you want to use as context.\n  6. Type a prompt and hit Enter to see first-token latency and tokens/sec metrics.\n\nHyper-IDE:\n  - Open HyperIDE-WASM.html to use the richer IDE view (Monaco editor + AI terminal).\n  - It shares the same llama.wasm and worker, so behaviour is consistent with Phase4-Ultra.html.\n
