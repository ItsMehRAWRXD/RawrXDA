# Extension Graveyard — Consolidation Map

**Goal:** 9 overlapping Electron/Node extensions → 1 RawrXD.exe (MASM64). Eliminate RAM bloat and `undefined_publisher` security risk.

**Environment: Windows only.** This consolidation targets the Win32 IDE, MASM64 modules, Windows Registry, `.cmd`/`.reg` launchers, and VS Code/Cursor on Windows. Do not run these scripts or expect these paths on Linux/macOS.

---

## The Fracture

| Family | What | Bloat |
|--------|------|--------|
| `cursor-*` | Qt/WebView wrappers around Ollama | 150MB+ each |
| `bigdaddyg-*` | Brand split across copilot/asm/chat | 4 extensions |
| `rawrz-*` / `rawrxd-*` | Survivors (LSP, etc.) | Native path |
| `undefined_publisher` | Local debug builds | **Security risk** (unsigned) |

## cursor-ollama-proxy — What to Salvage

**Location (harvested):** `harvested/undefined_publisher_cursor-ollama-proxy_extension.js`

**Logic worth keeping (already covered by native code):**

| Proxy behavior | Native equivalent |
|----------------|-------------------|
| Target `http://localhost:11434` | `RawrXD_HttpClient_Real.asm`: `wLocalhost`, port 11434 |
| Path rewrite `/v1/chat/completions` → `/api/chat` | Not needed: native calls Ollama directly |
| Path rewrite `/v1/models` → `/api/tags` | Not needed: use `/api/tags` in MASM |
| Wrote `cursor.chat.openaiBaseUrl` = `http://localhost:8081/v1` | RawrXD uses `ollama.baseUrl` = `http://localhost:11434` (no proxy) |

**Optional: Extension Host Hijacker (beacon-style).** To replace VS Code/Cursor’s Extension Host *in-process* with RawrXD native code (no JS), build and run the injector (requires admin):

- **Build:** `extension-host\Build_ExtensionHost_Hijacker.ps1` → `extension-host\build\RawrXD_ExtensionHost_Hijacker.exe`
- **Run:** Execute the .exe as Administrator; it finds Code Helper / Cursor Helper, allocates and copies a PIC payload, and starts RawrXD extension threads (ASM completion, LSP bridge) inside the host. See `extension-host\README.md`.

**Conclusion:** No code salvage required. The proxy existed only to translate **OpenAI-shaped** Cursor requests to Ollama. RawrXD talks to Ollama natively:

- **GET** `/api/tags` — list models (`RawrXD_HttpClient_Real.asm`: `wOllamaTagsPath`)
- **POST** `/api/chat` — chat (streaming) (`wOllamaChatPath`)
- **POST** `/api/generate` — legacy generate (`wOllamaGenPath`)

Merge checklist:

- [x] Default host/port: localhost:11434 (already in `RawrXD_HttpClient_Real.asm`, Win32IDE `m_ollamaBaseUrl`)
- [ ] Wire `ExtensionHandleChat` in `extension-host/RawrXD_Copilot.asm` to `Http_PostStream` (or C++ `ModelConnection`) so Sidebar chat streams from Ollama without any extension

## Unified Architecture (Post-Purge)

```
RawrXD.exe (MASM64)
├── Sidebar (Activity Bar)     ← Replaces bigdaddyg-copilot UI
├── WinHTTP Pipeline           ← Replaces cursor-ollama-proxy (RawrXD_HttpClient_Real.asm)
├── LSP Client (Native)        ← Replaces rawrxd-lsp-client
├── Genesis Assembler          ← Replaces bigdaddyg-asm-ide
└── Inference Core             ← Replaces all cursor-* AI extensions
```

**Unified manifest:** `src/RawrXD_Unified_Manifest.asm` maps all 9 extension IDs to Genesis symbols (OllamaGenerate, AgentOrchestrate, LSP_Initialize, GenesisMain, AssembleFile, etc.). Generated by `ExtensionExterminator.ps1` Phase 3.

## Security Note

`undefined_publisher` extensions are **unsigned debug builds**. Remove them from all hosts; use only signed or locally trusted native DLLs/ASM that RawrXD loads explicitly.

**Status:** 7–9 extensions → 1 executable. Hammer falls when you run the purge and wire `ExtensionHandleChat` to the WinHTTP pipeline.
