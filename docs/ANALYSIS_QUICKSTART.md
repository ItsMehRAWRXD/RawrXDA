# RawrXD Model Analysis — 15-Minute Quickstart

**Purpose:** Load any GGUF model, list every tensor (autopsy), and compare two models (neurological diff). All output streams to the terminal. Production-ready for tuning and hotpatch workflows.

---

## 1. One-shot implant (no surgery)

From repo root:

```powershell
.\scripts\Implant_Model_Analysis.ps1
```

This:

- Verifies analysis sources (`model_anatomy`, `neurological_diff`, `model_analysis_cli`)
- Runs Vulkan + analysis auto-wire (finds SDK, generates `analysis.ical.json`)
- Does **not** build by default

To also build:

```powershell
.\scripts\Implant_Model_Analysis.ps1 -Build
```

---

## 2. Build only (if already implanted)

Ensure Vulkan SDK is installed and wired:

```powershell
.\scripts\RawrXD_Vulkan_Analysis_AutoWire.ps1 -ProjectRoot . -GenerateIcal
```

Then build the ModelAnalysis CLI:

```powershell
# If using BUILD_ORCHESTRATOR
.\BUILD_ORCHESTRATOR.ps1 -Mode quick

# Or CMake directly
cmake --build build --target RawrXD-ModelAnalysis
```

Output: `build\bin\RawrXD-ModelAnalysis.exe` (or `bin\RawrXD-ModelAnalysis.exe` depending on config).

---

## 3. Autopsy — list every tensor (stream to terminal)

```powershell
.\build\bin\RawrXD-ModelAnalysis.exe --autopsy "C:\path\to\model.gguf"
```

Human-readable lines stream to stdout. Optional JSON:

```powershell
.\build\bin\RawrXD-ModelAnalysis.exe --autopsy "C:\path\to\model.gguf" --json
```

Use this to see layer/tensor names, types, sizes, and categories (embedding, attention_q/k/v/o, ffn_gate/up/down, norm, output, misc) for hotpatch/IDE integration.

---

## 4. Neurological diff — compare two models

Same prompt, two models; see where they differ (tensor only-in-A, only-in-B, size/type/shape mismatch):

```powershell
.\build\bin\RawrXD-ModelAnalysis.exe --diff "model_base.gguf" "model_finetuned.gguf"
```

JSON:

```powershell
.\build\bin\RawrXD-ModelAnalysis.exe --diff "model_base.gguf" "model_finetuned.gguf" --json
```

Use for before/after hotpatch verification or base vs fine-tuned comparison.

---

## 5. CLI space streaming

All output is stdout. Pipe or redirect as needed:

```powershell
.\build\bin\RawrXD-ModelAnalysis.exe --autopsy model.gguf --json | Out-File manifest.json
.\build\bin\RawrXD-ModelAnalysis.exe --diff A.gguf B.gguf --json | jq .
```

---

## 6. analysis.ical.json

Generated by `RawrXD_Vulkan_Analysis_AutoWire.ps1 -GenerateIcal`. Lists Vulkan SDK path and analysis targets (e.g. RawrXD-ModelAnalysis with `stream_to_terminal: true`) for tooling and IDE integration.

---

## 7. Troubleshooting

| Issue | Action |
|-------|--------|
| Vulkan not found | Install [Vulkan SDK](https://vulkan.lunarg.com/), then run `.\scripts\RawrXD_Vulkan_Analysis_AutoWire.ps1 -GenerateIcal` |
| Parse error / failed to open | Ensure path to GGUF is correct and file is not locked |
| Build fails on vulkan.h | Set `VULKAN_SDK` or run AutoWire so `find_package(Vulkan)` succeeds |

## 8. Use from Win32 IDE (CMake)

When building the Win32 IDE, `RawrXD-ModelAnalysisLib` is linked and `RAWR_HAS_MODEL_ANATOMY` is defined. You can then call from C++:

- **`getModelAnatomyJson(bool pretty)`** — Returns the full tensor manifest (JSON) for the currently loaded model, or empty string if none.
- **`getModelDiffJson(pathA, pathB, bool pretty)`** — Opens the two GGUF files, builds anatomies, diffs them, and returns JSON.

Use these from menus, agentic bridge, or output panels to expose anatomy/diff in the IDE without spawning the CLI.
