# Changelog

All notable changes to this project will be documented in this file.

## [Unreleased]

### Fixed
- **Electron AI providers:** `localhost` → **`127.0.0.1`** in `config/providers.json`, `preferIpv4Loopback()` on every request, and **`http.Agent({ family: 4 })`** for axios so Node cannot prefer **`::1`** when Ollama is IPv4-only; `providers.json` cache cleared on load. Clearer **`ECONNREFUSED`** text (`docs/AUTONOMOUS_AGENT_ELECTRON.md`).
- **Electron IRC bridge:** `electron/irc_protocol.js` — repaired invalid **`split(/\n')`** typo that broke module load; protocol helpers + **`irc-bridge-selfcheck.js`** validate chunking and parsers without opening sockets.
- **Win32 IDE launch:** Register **`__pfnDliFailureHook2`** → **`DelayLoadFailureHook`** so missing **delay-loaded** DLLs (e.g. **`vulkan-1.dll`**, **`D3DCOMPILER_47.dll`**) show a **message box** instead of an immediate exit with **no window**. See **`docs/WIN32_IDE_WONT_LAUNCH.md`**.
- **Integrated runtime boot:** `IntegratedRuntime::boot()` lived in `runPhase("integrated_runtime")` but that phase was **not** listed in GUI **`heavyPhases`** or **`config/startup_phases.txt`**, so `TranscendenceCoordinator::initializeAll()` often never ran at startup and `IntegratedRuntime::shutdown()` was a no-op. Added **`integrated_runtime`** after **`showWindow`** in those sequences and in **`startup_phase_registry.cpp`** defaults. Details: **`docs/IDE_LAUNCH_AUDIT_FINDINGS.md`**.
- **GGUF `readTensorData`:** when the full checkpoint is resident at **`mappedData`** (mmap / `MapViewOfFile` / small heap mirror), tensor payloads are **`memcpy`’d** from **`mappedData + abs_offset`** instead of **`seek`+read** on `NativeFile` (bounds-checked vs `modelFileSize`).
- **GGUFRunner:** Log **`blk.* weights loaded`** / **`inference_ready=yes|no`** after **`parseGgufLayerWeights`**, or a **warning** when layer load fails (missing tensors, unsupported quant, GQA/FFN mismatch) for easier TpsSmoke / batch diagnosis.
- **GGUF tensor offsets:** `parseGgufTensorTable()` now matches ggml/llama.cpp: tensor `offset` in metadata is **relative** to **`align_up(end_of_tensor_info, general.alignment)`** (default **32**). Stored **`tensorDataBaseOffset`** + per-tensor **absolute** `seek` positions fix wrong reads / silent garbage that could **AV** on SIMD paths. **`general.alignment`** is read in `ingestGgufKvMetadata()` into **`ggufTensorAlignment`** when present.
- **GGUF metadata:** `ingestGgufKvMetadata()` reads **llama.block_count**, **embedding_length**, **vocab_size**, head counts, **general.architecture** from real GGUF KV (not the brittle 8 MiB ASCII substring scan). **`sanitizeArchMetadata()`** caps insane layer/embed/vocab/head counts to avoid KV-cache **size_t** overflow → **0xC0000005**.
- **GGUF tensor table:** reject absurd tensor **name_len** before `read()` to avoid corrupt streams allocating huge buffers.
- **GGUF tensor data base:** `parseGgufTensorTable` maps **relative** tensor offsets to **absolute** file offsets via aligned `tensorDataBaseOffset`; **`general.alignment`** is now read from KV (default 32) for that padding instead of a fixed 32-byte bitmask.
- **TpsSmoke (Win32):** `SetUnhandledExceptionFilter` emits **`RAWRXD_TPS_JSON=`** with **`phase=native_crash`** and **NTSTATUS** in `detail` when the process still dies with an unhandled exception (so batch CSV is not stuck on **`no_json_line`** for **`-1073741819`**).
- **TpsSmoke / KV cache:** `setMaxTokens` is called **before** `loadModel` so `keyCache` / `valueCache` allocation uses the intended decode length (not the constructor default 64).
- **Benchmark-TpsSmoke-Models.ps1:** `Test-TpsSmokeLikelyCrashed` coerces string/null exit values and uses 64-bit math so **`-1073741819` (`0xC0000005`)** reliably maps to **`Skipped=crashed`** (not `no_json_line`).
- **GGUFRunner reload:** `loadGGUFModel` now calls `releaseWeightFileBacking()` before opening a new file and clears `layers` / embeddings / caches / `tensorTable` so stale per-layer weights cannot carry across models (avoids intermittent **0xC0000005**). `inferenceWeightsReady()` treats **`n_layers == 0`** as not ready (no longer returns true).
- **GGUFRunner / TpsSmoke:** `inferenceWeightsReady()` + early refusal — batch runs no longer hit **0xC0000005** when metadata reports `n_layers` but **no `blk.*` weights** are loaded; `RawrXD-TpsSmoke` exits **2** with phase `weights_incomplete` and still emits `RAWRXD_TPS_JSON=` when `RAWRXD_TPS_MACHINE_JSON=1`.
- **EditorOperations (Win32):** Undo/redo uses a consistent **applied-count** model with redo-branch truncation; **`ReplaceText`** no longer deadlocks on **`m_filesMutex`**; **`Cut`** avoids **`Copy()` + second lock** deadlock; **`Undo`/`Redo`** apply inverses without pushing nested stack entries; **multi-line cut** clears the undo stack (honest reset vs. fake history).
- **RouterOperations (Win32):** **Open / save / folder** dialogs use Vista+ **`IFileOpenDialog` / `IFileSaveDialog`** with **UTF-8** result paths; optional UTF-8 initial path for save via **`SHCreateItemFromParsingName`** (`shlobj.h`, COM apartment scope).
- **GitHub Actions:** **`.github/workflows/ci.yml`** — removed merge conflict markers (`<<<<<<<` / `>>>>>>>`); added **`master`** to **`push`** branch filters so CI runs on the mainline name.
- **GGUFRunner (Windows):** Large `.gguf` files no longer use a ~filesize `new float[]` heap mirror (OOM/crash). `CreateFileMappingW` + `MapViewOfFile` read-only mapping; heap copy capped at **512 MiB** fallback. `releaseWeightFileBacking()` unmaps/closes Win32 handles; `#undef Logging` after `windows.h` fixes MSVC treating `Logging` as a macro.
- **Local HTTP server — model bridge JSON:** `Win32IDE_LocalServer.cpp` now implements `GET /api/model/profiles`, `POST /api/model/load`, `POST /api/model/unload`, and `GET /api/engine/capabilities` (aligned with `gui/ide_chatbot_engine.js` and the legacy `tool_server` MASM bridge). Fixes `fetch().json()` / Dual-Agent flows that previously hit `Unknown endpoint` or non-JSON responses. `buildHttpResponse` now emits correct status lines for **422** and **500**.
- **Win32IDE.h Windows headers:** Correct include order for networking + shell dialogs — `winsock2.h` → `ws2tcpip.h` → `windows.h` → `commdlg.h` (with `WIN32_LEAN_AND_MEAN`), fixing `commdlg.h` before `windows.h` / Winsock-after-Windows issues that could trigger cascading compile errors.
- **Metrics Collector**: Resolved recursive mutex locking in `metrics_collector.cpp` by using snapshot-based aggregation.
- **Feature Tests**: Added VRAM allocation guards in `production_feature_test.cpp` to prevent crashes on systems with limited Vulkan budget.
- **MASM Entropy**: Implemented `XorShift64*` fast entropy generation in `soft_throttle_dispatch.asm` for high-performance thermal throttling jitter.
- **WindowManager (Win32):** **`Initialize()`** logs an **`IDELogger::WARNING`** that this secondary shell does **not** create the primary IDE frame (production window lives in **`Win32IDE_Core`**).
