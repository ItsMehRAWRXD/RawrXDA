# RawrXD v1.1 Community Roadmap

**Released:** Post v1.0.0 (March 2026+)  
**Status:** Planning / Low Priority  
**Community Input:** Welcome

---

## Philosophy

v1.0.0 delivers the **core IDE** - fully functional, production-ready, zero stubs. v1.1 adds **optional competitive features** that are nice-to-have but not critical to core functionality.

This roadmap prioritizes **high-ROI features** that enhance user experience without bloat.

---

## Priority Tiers

### P2: High-Value Enhancements (3-5 days effort)

These add meaningful capability with reasonable implementation cost.

#### 1. PDB Symbol Handler
**Status:** Design phase  
**Effort:** 2-3 days  
**Why:** Enable debugging of code generated by the inference engine

```cpp
// Proposed: handlePdbLoad
auto QuerySymbols(const char* pdbPath, uint64_t addr) 
  -> Optional<SymbolInfo>;

auto GetSourceLocation(const char* pdbPath, const StackFrame& frame)
  -> Optional<SourceLocation>;

auto ListPublicSymbols(const char* pdbPath)
  -> Vector<SymbolInfo>;
```

**Use Cases:**
- Step through generated code with symbol names
- Inspect local variables in inference output
- Correlate machine code to source functions
- Build profiler integration

**Implementation Plan:**
1. Link against `dbghelp.lib` (Windows SDK)
2. Load PDB on demand with caching
3. Resolve RVA → source file:line
4. Parse CodeView streams for variable types
5. Cache results in `.rawrxd_state\pdb_cache.json`

**Estimated Release:** v1.1.0 (May 2026)

---

#### 2. Voice-to-Code (STT) Handler
**Status:** Research phase  
**Effort:** 3-4 days  
**Why:** Hands-free coding for accessibility and demos

```cpp
// Proposed: handleVoiceStt
auto TranscribeAudio(const char* wavPath) 
  -> CommandResult;  // "write a fibonacci function"

auto StreamVoiceInput(int durationMs, const char* model)
  -> AsyncStream<std::string>;  // Live transcription

auto GetKeywordCommands() 
  -> Vector<std::string>;  // "compile", "debug", "search"
```

**Use Cases:**
- Accessibility for speech-impaired developers
- Quick voice search in large codebases
- Demo mode (impressive party trick 🎤)
- Hands-free testing during builds

**Implementation Plan:**
1. Use Windows.Media.SpeechRecognition API
2. Or integrate Whisper.cpp (lightweight, offline)
3. Send transcription to inference for command parsing
4. Log audio + transcription for audit
5. Add confidence scoring for keyword commands

**Dependencies:**
- Either WinRT APIs (built-in) OR
- Whisper.cpp + ONNX Runtime (~20MB)

**Estimated Release:** v1.1.1 (June 2026)

---

#### 3. Audit Trail Handler
**Status:** Requirements phase  
**Effort:** 2-3 days  
**Why:** Enterprise compliance & usage tracking

```cpp
// Proposed: handleAuditLog
auto LogCommand(const CommandContext& ctx, const CommandResult& result)
  -> bool;  // Record to audit trail

auto GetAuditTrail(const char* startTime, const char* endTime,
                   const char* filterUser)
  -> Vector<AuditEntry>;

auto ExportAuditLog(const char* format)  // "csv", "json", "syslog"
  -> FileHandle;
```

**Use Cases:**
- SOC 2 compliance (log all commands + results)
- Usage metrics (handlers called, success rates)
- Security events (failed auth, policy violations)
- Incident investigation (what happened when)

**Implementation Plan:**
1. Persist to structured log format (JSON Lines)
2. Include: timestamp, user, handler, args, result, duration
3. Rotate logs when > 100MB
4. Support syslog export for SIEM integration
5. Encrypt sensitive args (passwords, tokens)

**Retention Policy:**
- Hot logs: 30 days (.rawrxd_state/audit/)
- Archive: 1 year (if configured)

**Estimated Release:** v1.1.2 (June 2026)

### 🚫 Won't Fix (Scope Creep Prevention)

These are **explicitly rejected** to maintain focus:

| Feature | Why Not | Blocker |
|---------|---------|---------|
| **WebView2 Integration** | Adds 100MB, licensing complexity | Bloat + licensing |
| **Electron Wrapper** | Defeats purpose of native IDE | Anti-goal |
| **Cloud Tenancy** | Enterprise scope, mega cost | Out of scope |
| **Plugin Marketplace** | Distribution, security QA | Future (v2.0+) |
| **Git Integration** | Delegated to CLI tools | Use git CLI |
| **Docker/Container UI** | Orthogonal to IDE | Separate tool |

## FAQ

**Q: Why not include these in v1.0?**  
A: Scope cap. These are nice-to-have. v1.0 is the "core IDE" release.

**Q: Will voice STT work offline?**  
A: Yes, with Whisper.cpp. Or online with Azure Speech Services.

**Q: Can I use my own LLM for audit analysis?**  
A: Yes! Audit logs are standard JSON—pipe them to any tool.

**Q: What about mobile / web version?**  
A: Not planned. RawrXD is native-first. Web is v3.0+ at earliest.
